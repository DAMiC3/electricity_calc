# Food Ordering App - Tech Stack

## Frontend (Mobile App)
- **Technology:** Flutter
- **Reason:** Cross-platform mobile development (iOS & Android from one codebase)
- **Installation:** [Flutter Windows Install Guide](https://docs.flutter.dev/get-started/install/windows)

## Backend (Server)
- **Technology:** Python + FastAPI
- **Reason:** Modern, fast, easy to use API framework
- **Installation:** 
  - Python from [python.org](https://www.python.org/downloads/)
  - FastAPI: `pip install fastapi uvicorn`

## Database
- **Development:** SQLite
  - **Reason:** Built into Python, no setup required, good for development
- **Production:** PostgreSQL
  - **Reason:** Industry standard, scalable, works on Windows

## Hosting
- **Backend:** Render.com, Heroku, or Railway (all have free tiers)
- **Frontend:** Google Play Store and Apple App Store

## Authentication
- **Technology:** Firebase Authentication
- **Reason:** Free, easy setup, supports multiple sign-in methods

## Push Notifications
- **Technology:** Firebase Cloud Messaging (FCM)
- **Reason:** Free, integrates well with Flutter
